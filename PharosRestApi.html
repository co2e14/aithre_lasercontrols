PHAROS Supervisor REST API documentation version v0

http://LASER_IP_ADDRESS:20022/v0

Introduction

PHAROS Supervisor greatly simplifying external control of the PHAROS laser. This REST server is not almighty, though; it will only accept commands that corresponds to PHAROS User App logic.

Since this API a HTTP-based service, consuming it is easy in pretty much any platform, being it LabView, .NET languages or Python.

Usage

PHAROS Supervisor API is split into parts: /Basic, /Advanced, /StretcherCompressor, /Burst. /Basic contains casual, often used, features, while /Advanced, as the name implies, targets more complicated stuff.

Any GET request can be issued using any browser, so, to read states, browser is all you need! However, to practice PUT and POST requests, we recommend using Postman. You can get it at www.getpostman.com. Although we do not use it internally, users also had great results with Fiddler.

Basic usage

Defines basic functionality. Maps with UI title screen on pretty much 1:1 basis.

/Basic get

get

Gets all basic properties that have GET methods. All of them can be queried individually, but sometimes it is more convenient to get them all in one batch. The list is not fixed; it tends to grow over time as new properties are added. Also, some properties may not be relevant for a particular laser model (for example, burst-related ones for a non-burst laser).

Successful operation (response code 200)

Operation successful. GETable properties are in the body of the response.

""
→

/Advanced/ActualStateId get

get

Gets actual state ID. This is a numerical representation of /Basic/ActualStateName2 endpoint. Possible values are:

Unknown = 0x00
StandingBy = 0x01
Service = 0x02
InFieldUpdate = 0x04
Initializing = 0x08
GoingToStandby = 0x10
Failure = 0x20
Housekeeping = 0x40
Operational = 0x80
ShuttingDown = 0x100
Off = 0x200
DetectingLaserState = 0x400
Disconnected = 0x800
OscOutput = 0x1000.

Successful operation (response code 200)

Operation successful. Actual state ID is in the body of the response.

 

request body example

""
→
response body example

"2048"

get

Gets actual oscillator output attenuator percentage between 0.0 and 100.0 where 0 mans that oscillator output is closed (as much as possible) and 100.0 means that oscillator output is fully open.

Successful operation (response code 200)

Operation successful.

/OscillatorOutput/TargetAttenuatorPercentage put get

put

Sets new target attenuator value. After new value is accepted, attenuator motor will start moving to this position. You should wait for ActualAttenuatorPercentage to match your new target value. Accepts value between 0.0 and 100.0. Laser must be in "OscOutput" state before you can set this value.

Successful operation (response code 200)

Operation successful.
